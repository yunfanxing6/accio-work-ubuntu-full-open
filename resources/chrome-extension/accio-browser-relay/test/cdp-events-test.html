CDP Events Test Page

Test all CDP event handlers in accio-browser-relay. Open Extension Service Worker DevTools to see [evt:*] logs.

1. beforeunload Dialog

BLOCKING — auto-accept

Enables beforeunload handler. After clicking "Enable", closing this tab will trigger "Leave site?" dialog → extension should auto-accept it.

2. alert() Dialog

BLOCKING — auto-accept

Triggers alert() which blocks until Page.handleJavaScriptDialog is called.

3. confirm() Dialog

BLOCKING — auto-accept

Triggers confirm(). Extension auto-accepts (returns true).

4. prompt() Dialog

BLOCKING — auto-accept

Triggers prompt() with default value. Extension accepts with defaultPrompt.

5. Navigation Events

Page lifecycle

Triggers various Page.frame* events. (frameNavigated, navigatedWithinDocument, loadEventFired, etc.)

6. window.open

Page.windowOpen

Opens a new window/popup → triggers Page.windowOpen event.

7. Download

Page.downloadWillBegin

Triggers a file download → Page.downloadWillBegin / downloadProgress events.

8. File Chooser

Page.fileChooserOpened

Opens file input → Page.fileChooserOpened (if Page.setInterceptFileChooserDialog enabled).

9. Console & Runtime

Runtime events

Triggers console API calls and exceptions (requires Runtime.enable via relay).

10. Security / SSL

Security events

Navigate to a page with bad cert → Security.certificateError. (Requires Security.enable + Security.setOverrideCertificateErrors)

11. debugger Statement

BLOCKING — auto-resume

Executes `debugger;` → triggers Debugger.paused event (requires Debugger.enable via relay). Extension auto-resumes.

12. Rapid Dialog Stress Test

BLOCKING × N

Fires multiple dialogs in sequence to test rapid auto-accept handling.
