Enable Browser Connection

Your Chrome supports direct connection — no extension needed!

  1. Enable Remote Debugging
    Toggle the remote debugging switch to On.
    On Windows, Chrome may block this page from opening automatically. Copy and paste the URL into your address bar:
    chrome://inspect/#remote-debugging Click to copy
    How it works
    Chrome M144+ includes a built-in remote debugging interface. Accio connects directly to Chrome via the DevTools Protocol (CDP) — faster and simpler than using an extension.
  2. Allow Connection
    When Accio requests a debugging session, Chrome will show a permission dialog. Click Allow to let Accio connect.
    About the automation banner
    While Accio is connected, Chrome displays "Chrome is being controlled by automated test software". This is a standard security notice confirming the debugging session is active.
  3. You're All Set
    Accio can now directly control your browser tabs. No extension installation required.
    When do I need to reconnect?
    You need to click "Allow" again each time you restart Accio or Chrome. During normal use, the connection stays active automatically — no repeated approval needed.
    Security
    Remote debugging only works on localhost (127.0.0.1). Each new session requires explicit user permission via the Chrome dialog. You can disable it at any time by toggling the switch off.
